# 네트워크 상황과 Tailscale 연결 메모

> duhyunkim.page를 라즈베리파이(k3s)에 호스팅하면서, Mac에서 Pi로 원격 접속하려다
> 마주친 네트워크 문제와 그 해법(Tailscale)을 기록한다. 나중에 다시 볼 용.

## 한 줄 결론

집의 ISP 장비가 Mac과 Pi에게 **서로 다른 공인 서브넷**의 IP를 줘서 로컬 직접 통신이
원천 불가능했고, **Tailscale**(인터넷 위에 까는 암호화 가상 랜선)으로 해결했다.

## 진단: 왜 SSH가 안 됐나

토폴로지는 `ISP 모뎀 → 랜선 허브 → Mac / Pi 각각`. 그런데 받은 IP가 이랬다:

| | IP | 게이트웨이 | 서브넷 |
|---|---|---|---|
| Mac | `210.100.239.185` | `210.100.239.1` | `210.100.239.0/24` |
| Pi  | `210.90.128.229`  | `210.90.128.1`   | `210.90.128.0/24`  |

- 모뎀이 NAT를 안 하고 **공인 IP를 그대로** 흘림 (Mac이 사설이 아닌 공인 IP를 받은 게 단서).
- 그런데 Mac과 Pi가 **완전히 다른 공인 /24**에, 게이트웨이도 다름.
- → 같은 허브에 꽂혀 있어도 둘은 사실상 **다른 네트워크**에 있는 셈. 로컬에서 서로 못 봄.
- 그래서 `ssh`, `ping raspberrypi.local`(mDNS), ARP 탐색 전부 실패. **장비/망 문제이지 설정 실수가 아님.**

증거 요약: Mac↔Pi 직접 통신 ✗ / 양쪽 다 인터넷은 ✓ / Pi는 Raspberry Pi Connect로 Online.

### 지금 상황 (물리)

```
                          ☁  인터넷
                          │
                  ┌───────┴────────┐
                  │   ISP 모뎀      │   NAT 안 함 → 공인 IP를 그대로 흘림
                  └───────┬────────┘
                          │
                  ┌───────┴────────┐
                  │    랜선 허브     │   ← 각자 다른 공인 IP를 받음
                  └──┬──────────┬──┘
                     │          │
          ┌──────────┘          └──────────┐
     ┌────┴──────┐                    ┌────┴──────┐
     │   Mac     │ ◄─ ─ ✗ 차단 ✗ ─ ─► │    Pi     │
     │ 210.100.  │   서로 직접 못 봄    │ 210.90.   │
     │ 239.185   │                    │ 128.229   │
     └───────────┘                    └───────────┘
       (다른 공인 /24)                   (다른 공인 /24)
            └───────── 둘 다 인터넷은 OK ─────────┘
```

## 해법: Tailscale

### 무엇인가
- **WireGuard 기반 VPN.** 내 기기들끼리만 보이는 사설 네트워크(**tailnet**)를 만든다.
- 각 기기에 가상 랜카드(`tailscale0`)와 고정 사설 IP(`100.x.x.x`)가 생긴다.
- 연결이 **바깥(인터넷)으로 먼저 나가서 만나는** 구조라 NAT·클라이언트 격리·다른 서브넷과 무관하게 통한다.
- **Tailscale SSH**를 켜면 키 관리 없이 `ssh`가 된다. 개인용 무료.

### 적용 후 (논리)

```
                  Tailscale 좌표 서버 (Control)
                  · tailnet 명단 관리
                  · 공개키/접속경로만 중개 (데이터 내용은 안 봄)
                       ▲                    ▲
              로그인   │                    │  로그인
                       │                    │
            ┌──────────┘                    └──────────┐
       ┌────┴──────┐                            ┌───────┴───┐
       │   Mac     │════ WireGuard 암호화 터널 ═══│    Pi     │
       │ 100.x.x.1 │     (인터넷으로 P2P 직결)     │ 100.x.x.2 │
       └───────────┘                            └───────────┘
```

```
물리:   Mac ─허브─ 모뎀 ─🌐─ … ─🌐─ 모뎀 ─허브─ Pi   (서로 못 봄)
논리:   Mac ═══════════ 가상 직결 터널 ═══════════ Pi   (100.x ↔ 100.x)
```

### 작동 순서
1. **로그인** — 양쪽 `tailscale up`이 같은 계정/ tailnet에 등록.
2. **발견** — 좌표 서버가 서로의 공개키·경로만 중개(데이터는 안 봄).
3. **직결 터널** — 둘이 WireGuard 암호화 터널을 인터넷 너머로 직접 맺음. 둘 다 공인 IP라 P2P 직결이 잘 됨(실패 시 Tailscale 릴레이로 자동 우회, 암호화 유지).

## 설정 명령 (한 일)

Pi (Raspberry Pi Connect 브라우저 셸에서):
```sh
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up --ssh        # 출력된 로그인 URL로 인증
tailscale ip -4                # Pi의 100.x.x.x 확인
```

Mac:
```sh
# Tailscale 앱 설치 후 같은 계정으로 로그인
tailscale status               # Pi가 목록에 보이면 성공
ssh duhyunkim@<Pi-tailscale-IP 또는 호스트명>
```

- Pi 사용자/호스트명: `duhyunkim@duhyunkim-pi`

## 대안 (참고)
- 내 공유기를 모뎀 뒤에 두고 Mac·Pi를 그 LAN에 묶으면 로컬 통신이 정상화됨(물리적 해결).
- Cloudflare Tunnel로 SSH도 터널링 가능(이미 도메인이 Cloudflare라 생태계 통일). 단 설정은 Tailscale보다 약간 복잡.
